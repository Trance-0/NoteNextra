The repair problem

Main challenge:

  • Locality (number of contacted servers)
  • Bandwidth (number of bits transferred)

From last lecture we build optimal Local Recoverable Codes (LRCs) for storage systems.

Let C=[n,k]q\mathcal{C} = [n, k]_qC=[n,k]q​ which is rrr-LRC, with minimum distance ddd.

  • Bound 1: kn≤rr+1\frac{k}{n}\leq \frac{r}{r+1}nk​≤r+1r​.

  • Bound 2: d≤n−k−kr+2d\leq n-k-\frac{k}{r}+2d≤n−k−rk​+2.

  • Optimal LRC:

  • Let A={α1,…,αn}\mathcal{A} = \{\alpha_1, \ldots, \alpha_n\}A={α1​,…,αn​}, partition A\mathcal{A}A to Ai\mathcal{A}_iAi​ for i=1i=1i=1 to nr+1\frac{n}{r+1}r+1n​.

  • g∈Fq[x]g\in \mathbb{F}_q[x]g∈Fq​[x] is good if deg⁡(g)=r+1\deg(g) = r+1deg(g)=r+1 and ggg is constant on all Ai\mathcal{A}_iAi​‘s.

  • C={fa(αi)}i=1n∣a∈Fqk}\mathcal{C} = \{f_a(\alpha_i)\}_{i=1}^{n}|a\in \mathbb{F}_q^k\}C={fa​(αi​)}i=1n​∣a∈Fqk​}, where

    • fa(x)=∑i=0r−1fa,i(x)⋅xif_a(x) = \sum_{i=0}^{r-1} f_{a,i}(x)\cdot x^ifa​(x)=∑i=0r−1​fa,i​(x)⋅xi,

  • dim⁡C=k\dim \mathcal{C} = kdimC=k and d=n−k−kr+2d = n-k-\frac{k}{r}+2d=n−k−rk​+2.

Minimizing the repair bandwidth

Goal: understand repair bandwidth.

  • What is the minimum repair bandwidth?
  • Is repair bandwidth in trade-off with other parameters?
  • Tool: The information flow graph.

Spoiler alert:

  • Tradeoff: Storage Repair and bandwidth.
  • Codes which achieve an optimal tradeoff.

Information flow graph

We can model the repair problem as a directed graph.

  • Source: System admin.
  • Sink: Data collector.
  • Nodes: Storage servers.
    • Nodes leave/crash
    • Newcomer replaces them →\to→ new nodes.
  • Edges: Represents transmission of information. (Number of Fq\mathbb{F}_qFq​ elements is weight.)

Main observation:

  • kkk elements from Fq\mathbb{F}_qFq​ must “flow” from the source (system admin) to the sink (data collector).
  • Any cut (U,U‾)(U,\overline{U})(U,U) which separates source from sink must have capacity at least kkk.

Roadmap:

Information flow graph →\to→ Minimum cut analysis →\to→ Bound on file size →\to→ Storage/bandwidth tradeoff.

Basic definitions for information flow graph

Warning

This is not the same as definitions in linear codes. kkk is not the dimension of the code and ddd is not the minimum distance of the code for general cases.

Parameters:

  • nnn is the number of nodes in the initial system (before any node leaves/crashes).
  • kkk is the number of nodes required to reconstruct the file kkk.
  • ddd is the number of nodes required to repair a failed node.
  • α\alphaα is the storage at each node.
  • β\betaβ is the edge capacity for repair.
  • BBB is the file size.

Goal: Find the trade off between n,k,d,α,β,Bn,k,d,\alpha,\beta,Bn,k,d,α,β,B using min-cut analysis of the information flow graph.

Initial system:

We denote the system admin as SSS

Sever as 1,2,…,n1,2,\ldots,n1,2,…,n, each with edge capacity α\alphaα.

For each new server:

We have two nodes ininin and outoutout, the edge weight is α\alphaα.

Connect to ddd previous nodes outoutout‘s with edge capacity β\betaβ.

Data collector:connects to kkk arbitrary nodes with each edge capacity α\alphaα.

Observe that:

  • File size BBB.
  • Any cut separating SSS form DCDCDC must have capacity at least BBB.
  • Otherwise, two different files are indistinguishable to DCDCDC.

Bound on bandwidth

Claim: mincut≥∑i=1k−1min⁡{(d−i)β,α}mincut\geq \sum_{i=1}^{k-1}\min\{(d-i)\beta, \alpha\}mincut≥∑i=1k−1​min{(d−i)β,α}.

Intuition: Let (U,U‾)(U, \overline{U})(U,U) be a cut separating SSS form DCDCDC.

  • The DC contacts newest kkk nodes, say n1,n2,…,nkn_1,n_2,\ldots,n_kn1​,n2​,…,nk​.
  • The cut must decide if to cross α\alphaα edges or β\betaβ edges.
  • At east d−id-id−i edges to go to UUU.

Proof

Let (U,U‾)(U, \overline{U})(U,U) be a cut separating SSS form DCDCDC, assuming S∈US\in US∈U and DC∈U‾DC\in \overline{U}DC∈U.

Every directed acyclic graph has topological sort.

Let xout1x_{out}^1xout1​ be the first outoutout node in U‾\overline{U}U. There are two cases:

  • xin1∈Ux_{in}^1\in Uxin1​∈U. Then α\alphaα edges must be crossed.
  • xin1∈U‾x_{in}^1\in \overline{U}xin1​∈U. Then all ddd incoming edges to xin1x_{in}^1xin1​ must be crossed. (Otherwise, there exists an earlier outoutout node xoutjx_{out}^jxoutj​ with xinj∈Ux_{in}^j\in Uxinj​∈U, contradicting the topological sort.)

So xout1x_{out}^1xout1​ contributes at least min⁡{dβ,α}\min\{d\beta, \alpha\}min{dβ,α} to the cut capacity.

Let xout2x_{out}^2xout2​ be the second outoutout node in U‾\overline{U}U. There are two cases:

  • xin2∈Ux_{in}^2\in Uxin2​∈U. Then α\alphaα edges must be crossed.
  • xin2∈U‾x_{in}^2\in \overline{U}xin2​∈U. Then at least d−1d-1d−1 incoming edges (111 edge may come from xout1x_{out}^1xout1​) to xin2x_{in}^2xin2​ must be crossed. (Otherwise, there exists an earlier outoutout node xoutjx_{out}^jxoutj​ with xinj∈Ux_{in}^j\in Uxinj​∈U, contradicting the topological sort.)

So xout2x_{out}^2xout2​ contributes at least min⁡{(d−1)β,α}\min\{(d-1)\beta, \alpha\}min{(d−1)β,α} to the cut capacity.

By repeating this process, we can show that the minimum cut capacity is at least ∑i=1k−1min⁡{(d−i)β,α}\sum_{i=1}^{k-1}\min\{(d-i)\beta, \alpha\}∑i=1k−1​min{(d−i)β,α}.

Storage/bandwidth tradeoff

Claim: There exists an information graph with mincut=∑i=1k−1min⁡{(d−i)β,α}mincut = \sum_{i=1}^{k-1}\min\{(d-i)\beta, \alpha\}mincut=∑i=1k−1​min{(d−i)β,α}.

Homework: Build this graph as follows:

  • Construct the initial graph with nnn nodes and the system admin.
  • Add n+kn+kn+k nodes, each node connects to the most recent ddd nodes.
  • Find the minimum cut capacity.

Corollary: B≤∑i=1k−1min⁡{(d−i)β,α}B\leq \sum_{i=1}^{k-1}\min\{(d-i)\beta, \alpha\}B≤∑i=1k−1​min{(d−i)β,α}.

Definition of regenerate code

A code which attains B=∑i=1k−1min⁡{(d−i)β,α}B=\sum_{i=1}^{k-1}\min\{(d-i)\beta, \alpha\}B=∑i=1k−1​min{(d−i)β,α} is called a regenerate code.

Goal: Find tradeoff between storage α\alphaα to repair bandwidth dβd\betadβ.

Let γ=dβ\gamma = d\betaγ=dβ, then B≤∑i=0k−1min⁡{(1−i/d)γ,α}B \leq \sum_{i=0}^{k-1}\min\{(1-i/d)\gamma, \alpha\}B≤∑i=0k−1​min{(1−i/d)γ,α}.

Tool: Fix γ\gammaγ and ddd, and minimize for α\alphaα (not shown).

Result: The storage/bandwidth tradeoff.

  • Each point on/above the line is feasible.
  • Points on the line = regenerating codes.
  • One endpoint: Minimum Bandwidth Regenerating (MBR) codes.
  • another endpoint: Minimum Storage Regenerating (MSR) codes

For Minimum Storage Regenerating (MSR) codes, we have α=Bk\alpha = \frac{B}{k}α=kB​, β=Bk(d−k+1)\beta = \frac{B}{k(d-k+1)}β=k(d−k+1)B​

For Minimum Bandwidth Regenerating (MBR) codes, we have α=dBkd−k(k−1)2\alpha = \frac{dB}{kd-\frac{k(k-1)}{2}}α=kd−2k(k−1)​dB​, β=Bkd−k(k−1)2\beta = \frac{B}{kd-\frac{k(k-1)}{2}}β=kd−2k(k−1)​B​

Notes:

  • In MSR α=B/k\alpha=B/kα=B/k, Data collector contacts kkk nodes and downloads B/kB/kB/k from each to reconstruct the file of size BBB, that is optimal.
  • In MBR β=B/(kd−k(k−1)2)\beta=B/(kd-\frac{k(k-1)}{2})β=B/(kd−2k(k−1)​), new comer download exactly what it stores, which is the same as replication. This has much smaller storage overhead in the replication.

Regenerating codes, Magic #1:

  • MBR: Same repair-bandwidth as replication (α\alphaα), at lower storage costs.
  • MSR: Same reconstruction-bandwidth (B/kB/kB/k) as replication, at lower storage costs.

Regenerating codes, Magic #2:

  • In MSR: γ=dβ=dBk(d−k+1)\gamma = d\beta = \frac{dB}{k(d-k+1)}γ=dβ=k(d−k+1)dB​, α=Bk\alpha = \frac{B}{k}α=kB​
  • In MBR: γ=dβ=dBkd−k(k−1)2\gamma = d\beta = \frac{dB}{kd-\frac{k(k-1)}{2}}γ=dβ=kd−2k(k−1)​dB​, α=dBkd−k(k−1)2\alpha = \frac{dB}{kd-\frac{k(k-1)}{2}}α=kd−2k(k−1)​dB​
  • Both decreasing functions of ddd.
  • ⇒\Rightarrow⇒ Less repair-bandwidth by contacting more nodes, minimized at d=n−1d = n - 1d=n−1.

Constructing Minimum bandwidth regenerating (MBR) codes from Maximum distance separable (MDS) codes

Observation: For MBR code with parameters n,k,dn, k, dn,k,d and β=1\beta = 1β=1, one can construct MBR with parameters n,k,dn, k, dn,k,d and any β\betaβ.

Next: Construct MBR for [n,k,d=n−1][n, k, d = n - 1][n,k,d=n−1] and β=1\beta = 1β=1.

In any MBR: α,β=dBkd−k(k−1)2,Bkd−k(k−1)2\alpha, \beta = \frac{dB}{kd-\frac{k(k-1)}{2}}, \frac{B}{kd-\frac{k(k-1)}{2}}α,β=kd−2k(k−1)​dB​,kd−2k(k−1)​B​

Specifically:

  • Storage α=dβ=d=n−1\alpha = d\beta = d = n - 1α=dβ=d=n−1.
  • File size B=kd−(k2)β=kd−(k2)B = kd - \binom{k}{2}\beta = kd - \binom{k}{2}B=kd−(2k​)β=kd−(2k​)

Take an [(n2),B][\binom{n}{2}, B][(2n​),B] MDS code (e.g., Reed-Solomon).

Need q≥n2q\geq \frac{n}{2}q≥2n​.

Consider a complete graph KnK_nKn​ on nnn nodes.

  • (n2)\binom{n}{2}(2n​) edges.
  • Place each codeword symbol on a distinct edge.
  • Storage server iii stores all codeword symbols adjacent with node iii.
    • α=n−1\alpha = n - 1α=n−1.

Reconstruction on MBR codes

We use [(n2),B]q[\binom{n}{2}, B]_q[(2n​),B]q​ MDS code. So any BBB symbols suffice to reconstruct the file.

Any ttt nodes have (t2)\binom{t}{2}(2t​) edges between them, and (n−1)t−2(t2)(n-1)t-2\binom{t}{2}(n−1)t−2(2t​) edges to other nodes.

Overall (n−1)t−(t2)(n-1)t-\binom{t}{2}(n−1)t−(2t​). For t=kt=kt=k, we get kd−(k2)=Bkd-\binom{k}{2}=Bkd−(2k​)=B.
