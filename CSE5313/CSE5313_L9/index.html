Explicit optimal codes

Explicit optimal codes?

  • Singleton, Sphere-packing provide restrictions.
  • Gilbert-Varshamov provides existence.

Are there explicit optimal codes? That is,

  • Easily (polynomial time) encodable, decodable.

Yes! This lecture:

– Gustave Solomon [1930-1996] (Reed-Solomon code) – Irving S. Reed [1923-2012]. – David E. Muller [1924-2008] (Reed-Muller code)

Using Polynomials over Fq\mathbb{F}_qFq​

Reed-Solomon code

Note

The fundamental theorem of algebra:

A polynomial of degree kkk has at most kkk roots.

We have two equivalent definitions of a Reed-Solomon code:

  • As polynomial evaluations.
  • As linear codes (from generator matrix)

Efficient decoding (use Euclidean algorithm)

Reed-Muller code

Reed-Solomon codes: Evaluations of univariate polynomials of deg ≤ k−1k-1k−1.

Reed-Muller codes: Evaluations of multivariate polynomials of deg ≤k−1\leq k-1≤k−1

Example:

f(x1,x2,x3)=x1x22+x1x3+x2+x2x33f(x_1,x_2,x_3)=x_1x_2^2+x_1x_3+x_2+x_2x_3^3f(x1​,x2​,x3​)=x1​x22​+x1​x3​+x2​+x2​x33​

RM(r,m)={(f(α1),…,f(α2m))∣αi∈F2m,deg⁡f≤r}RM(r,m)=\left\{(f(\alpha_1),\ldots,f(\alpha_2^m))|\alpha_i\in \mathbb{F}_2^m,\deg f\leq r\right\}RM(r,m)={(f(α1​),…,f(α2m​))∣αi​∈F2m​,degf≤r}

  • Length n=2mn = 2^mn=2m.
  • Minimum distance 2m−r2^{m-r}2m−r (not shown).
  • Dimension = # of free coefficients in a multilinear polynomial of degree at most rrr.

Coding for storage

Requirements/Challenges in Storage Systems

  1. Challenge 1: Reconstruction.
    • The data collector must be able to reconstruct the file, even if some are nonresponsive.
      • Minimize reconstruction bandwidth.
  2. Challenge 2: Repair.
    • The system must maintain data consistency.
    • Failed servers must be repaired:
      • By contacting few other servers (locality, due to geographical constraints).
      • By minimizing bandwidth.
  3. Challenge 3: Storage overhead.
    • Minimize space consumption.
      • Minimize redundancy.

Naive solution: Replication

Fragment the file X=(X1,…,Xk)X = (X_1, \ldots, X_k)X=(X1​,…,Xk​).

  • Size of XiX_iXi​ = Whatever fits in a storage server.

Hold rrr copies of each XiX_iXi​.

  • I.e., n=rkn = rkn=rk servers in the system.

Storage overhead?

  • nk=r\frac{n}{k} = rkn​=r.

Repair?

  • XiX_iXi​ fails
  • ≥r\geq r≥r failures is lost data.

Reconstruction?

  • Possible if any r−1r-1r−1 servers fail.
  • Impossible for some ≥r\geq r≥r failures.

Use codes to improve storage efficiency

Reconstruction?

  • Lecture 1: If dH(C)≥dd_H(\mathcal{C})\geq ddH​(C)≥d, every pattern of at most d−1d-1d−1 erasures is recoverable.

  • Idea: Treat unavailable servers as erasures.

Is this better/worse than replication?

  • Say we wish to reconstruct from any ≈n10\approx \frac{n}{10}≈10n​ servers.
  • What would be the redundancy in replication vs. coding?

Coding:

  • Can reconstruct file from any n−d+1≈910nn-d+1\approx \frac{9}{10}nn−d+1≈109​n servers.
  • Resulting overhead nk=nn−d+1≈109\frac{n}{k}=\frac{n}{n-d+1}\approx \frac{10}{9}kn​=n−d+1n​≈910​ (constant!).

Replication:

To reconstruct from any 910n\frac{9}{10}n109​n servers, need r−1≈110nr-1\approx \frac{1}{10}nr−1≈101​n

Repair?

  • Need low locality (repair by contacting few other servers).
  • Need low bandwidth (repair by downloading as few bits as possible).

Repair in a replicated system:

  • XiX_iXi​ fails ⇒\Rightarrow⇒ reconstruct from a different copy.
  • Locality 1.
  • Optimal bandwidth.

Repair in a coded system:

  • repair one YiY_iYi​ ≈\approx≈ Reconstruct the entire file.
    • Locality n−d+1n-d+1n−d+1, high bandwidth.
  • Much worse than replication.
