NP-completeness

PPP: Polynomial-time Solvable

PPP: Class of decision problems LLL such that there is a polynomial-time algorithm that correctly answers yes or not for every instance l∈Ll\in Ll∈L.

Algorithm ”AAA decides LLL”. If algorithm AAA always correctly answers for any instance l∈Ll\in Ll∈L.

Example:

Is the number nnn prime? Best algorithm so far: O(log⁡6n)O(\log^6 n)O(log6n), 2002

Introduction to NP

  • NP≠\neq= Non-polynomial (Non-deterministic polynomial time)
  • Let LLL be a decision problem.
  • Let lll be an instance of the problem that the answer happens to be “yes”.
  • A certificate c(l) for lll is a “proof” that the answer for lll is true. [lll is a true instance]
    • For canonical decision problems for optimization problems, the certificate is often a feasible solution for the corresponding optimization problem.

Example of certificates

  • Problem: Is there a path from sss to ttt
    • Instance: graph G(V,E),s,tG(V,E),s,tG(V,E),s,t.
    • Certificate: path from sss to ttt.
  • Problem: Can I schedule kkk intervals in the room so that they do not conflict.
    • Instance: l:(I,k)l:(I,k)l:(I,k)
    • Certificate: set of kkk non-conflicting intervals.
  • Problem: ISET
    • Instance: G(V,E),kG(V,E),kG(V,E),k.
    • Certificate: kkk vertices with no edges between them.

If the answer to the problem is NO, you don’t need to provide anything to prove that.

Useful certificates

For a problem to be in NP, the problem need to have “useful” certificates. What is considered a good certificate?

  • Easy to check
    • Verifying algorithm which can check a YES answer and a certificate in poly(l)poly(l)poly(l)
  • Not too long: [poly(l)poly(l)poly(l)]

Verifier Algorithm

Verifier algorithm is one that takes an instance l∈Ll\in Ll∈L and a certificate c(l)c(l)c(l) and says yes if the certificate proves that lll is a true instance and false otherwise.

VVV is a poly-time verifier for LLL is it is a verifier and runs in poly(∣l∣,∣c∣)poly(|l|,|c|)poly(∣l∣,∣c∣) time. (c=poly(l)poly(l)poly(l))

  • The runtime must be polynomial
  • Must check every problem constraint
  • Not always trivial

Class NP

NP: A class of decision problems such that exists a certificate schema ccc and a verifier algorithm VVV such that:

  1. certificate is poly(l)poly(l)poly(l) in size.
  2. V:poly(l)V:poly(l)V:poly(l) in time.

P: is a class of problems that you can solve in polynomial time

NP: is a class of problems that you can verify TRUE instances in polynomial time given a poly-size certificate

Millennium question

P⊆NPP\subseteq NPP⊆NP? NP⊆PNP\subseteq PNP⊆P?

P⊆NPP\subseteq NPP⊆NP is true.

Proof: Let LLL be a problem in PPP, we want to show that there is a polynomial size certificate with a poly-time verifier.

There is an algorithm AAA which solves LLL in polynomial time.

Certificate: empty thing.

Verifier: (l,c)(l,c)(l,c)

  1. Discard ccc.
  2. Run AAA on lll and return the answer.

Nobody knows the solution NP⊆PNP\subseteq PNP⊆P. Sad.

Class of problem: NP complete

Informally: hardest problem in NP

Consider a problem LLL.

  • We want to show if L⊆PL\subseteq PL⊆P, then NP⊆PNP\subseteq PNP⊆P

NP-hard: A decision problem LLL is NP-hard if for any problem K∈NPK\in NPK∈NP, K≤pLK\leq_p LK≤p​L.

LLL is at least as hard as all the problems in NP. If we have an algorithm for LLL, we have an algorithm for any problem in NP with only polynomial time extra cost.

MindMap:

K  ⟹  L  ⟹  sol(L)  ⟹  sol(K)K\implies L\implies sol(L)\implies sol(K)K⟹L⟹sol(L)⟹sol(K)

Lemma P=NPP=NPP=NP

Let LLL be an NP-hard problem. If L∈PL\in PL∈P, then P=NPP=NPP=NP.

Proof:

Say LLL has a poly-time solution, some problem KKK in NPNPNP.

For any K∈NPK\in NPK∈NP, NP⊂PNP\subset PNP⊂P, P⊂NPP\subset NPP⊂NP, then P=NPP=NPP=NP.

NP-complete: LLL is NP-complete if it is both NP-hard and L∈NPL\in NPL∈NP.

NP-optimization: LLL is NP-optimization problem if the canonical decision problem is NP-complete.

Claim: If any NP-optimization problem have polynomial-time solution, then P=NPP=NPP=NP.

Is P=NPP=NPP=NP?

  • Answering this problem is hard.
  • But for any NP-complete problem, if you could find a poly-time algorithm for LLL, then you would have answered this question.
  • Therefore, finding a poly-time algorithm for LLL is hard.

NP-Complete problem

Satisfiability (SAT)

Boolean Formulas:

A set of Boolean variables:

x,y,a,b,c,w,z,...x,y,a,b,c,w,z,...x,y,a,b,c,w,z,... they take values true or false.

A boolean formula is a formula of Boolean variables with and, or and not.

Examples:

ϕ:x∧(¬y∨z)∧¬(y∨w)\phi:x\land (\neg y \lor z)\land\neg(y\lor w)ϕ:x∧(¬y∨z)∧¬(y∨w)

x=1,y=0,z=1,w=0x=1,y=0,z=1,w=0x=1,y=0,z=1,w=0, the formula is 111.

SAT: given a formula ϕ\phiϕ, is there a setting MMM of variables such that the ϕ\phiϕ evaluates to True under this setting.

If there is such assignment, then ϕ\phiϕ is satisfiable. Otherwise, it is not.

Example: x∧y∧¬(x∨y)x\land y\land \neg(x\lor y)x∧y∧¬(x∨y) is not satisfiable.

A seminar paper by Cook and Levin in 1970 showed that SAT is NP-complete.

  1. SAT is in NP
    Proof:
    ∃\exists∃ a certificate schema and a poly-time verifier.
    ccc satisfying assignment MMM and vvv check that MMM makes ϕ\phiϕ true.
  2. SAT is NP-hard. we can just accept it has a fact.

How to show a problem is NP-complete?

Say we have a problem LLL.

  1. Show that L∈NPL\in NPL∈NP.
    Exists certificate schema and verification algorithm in polynomial time.
  2. Prove that we can reduce SAT to LLL. SAT≤pLSAT\leq_p LSAT≤p​L (NOT L≤pSATL\leq_p SATL≤p​SAT) Solving LLL also solve SAT.

3-CNF-SAT

3-CNF-SAT: where every clauses has exactly 3 literals.

is NP complete [not all version of them are, 2-CNF-SAT is in P]

Input: 3-CNF expression with nnn variables and mmm clauses in the form:

number of total literals: 3m3m3m

Output: An assignment of the nnn variables such that at least one literal from each clauses evaluates to true.

Note:

  1. One variable can be used to satisfy multiple clauses.
  2. xix_ixi​ and ¬xi\neg x_i¬xi​ cannot both evaluate to true.

Example: ISET is NP-complete.

Proof:

Say we have a problem LLL

  1. Show that ISET∈NPISET\in NPISET∈NP
    Certificate: set of kkk vertices: ∣S∣=k∈poly(g)|S|=k\in poly(g)∣S∣=k∈poly(g)
    Verifier: checks that there are no edges between them O(Ek2)O(E k^2)O(Ek2)
  2. ISET is NP-hard. We need to prove 3SAT≤pISET3SAT\leq_p ISET3SAT≤p​ISET
    • Construct a reduction from 3SAT3SAT3SAT to ISETISETISET.
    • Show that ISETISETISET is harder than 3SAT3SAT3SAT.

We need to prove ϕ∈3SAT\phi\in 3SATϕ∈3SAT is satisfiable if and only if the constructed GGG has an ISETISETISET of size ≥k=m\geq k=m≥k=m

Reduction mapping construction

We construct an ISET instance from 3−SAT3-SAT3−SAT.

Suppose the formula has nnn variables and mmm clauses

  1. for each clause, we construct vertex for each literal and connect them (for x∨¬y∨zx\lor \neg y\lor zx∨¬y∨z, we connect x,¬y,zx,\neg y,zx,¬y,z together)
  2. then we connect all the literals with their negations (connects xxx and ¬x\neg x¬x)

  ⟹  \implies⟹

If ϕ\phiϕ has a satisfiable assignment, then GGG has an independent set of size ≥m\geq m≥m,

For a set SSS we pick exactly one true literal from every clause and take the corresponding vertex to that clause, ∣S∣=m|S|=m∣S∣=m

Must also argue that SSS is an independent set.

Example: picked a set of vertices ∣S∣=4|S|=4∣S∣=4.

A literal has edges:

  • To all literals in the same clause: We never pick two literals form the same clause.
  • To its negation.

Since it is a satisfiable 3-SAT assignment, xxx and ¬x\neg x¬x cannot both evaluate to true, those edges are not a problem, so SSS is an independent set.

  ⟸  \impliedby⟸

If GGG has an independent set of size ≥m\geq m≥m, then ϕ\phiϕ is satisfiable.

Say that SSS is an independent set of mmm, we need to construct a satisfiable assignment for the original ϕ\phiϕ.

  • If SSS contains a vertex corresponding to literal xix_ixi​, then set xix_ixi​ to true.
  • If contains a vertex corresponding to literal ¬xi\neg x_i¬xi​, then set xix_ixi​ to false.
  • Other variables can be set arbitrarily

Question: Is it a valid 3-SAT assignment?

Your ISET SSS can contain at most 111 vertex from each clause. Since vertices in a clause are connected by edges.

  • Since SSS contains mmm vertices, it must contain exactly 111 vertex from each clause.
  • Therefore, we will make at least 111 literals form each clause to be true.
  • Therefore, all the clauses are true and ϕ\phiϕ is satisfied.

Conclusion: NP-completeness

  • Prove NP-Complete:
    • If NP-optimization, convert to canonical decision problem
    • Certificate, Verification algorithm
    • Prove NP-hard: reduce from existing NP-Complete problems
  • 3-SAT Problem:
    • Input, output, constraints
    • A well-known NP-Complete problem
    • Reduce from 3-SAT to ISET to show ISET is NP-Complete
