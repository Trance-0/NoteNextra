CSE347 Analysis of Algorithms (Lecture 1)

Greedy Algorithms

  • Builds up a solution by making a series of small decisions that optimize some objective.
  • Make one irrevocable choice at a time, creating smaller and smaller sub-problems of the same kind as the original problem.
  • There are many potential greedy strategies and picking the right one can be challenging.

A Scheduling Problem

You manage a giant space telescope.

  • There are nnn research projects that want to use it to make observations.
  • Only one project can use the telescope at a time.
  • Project pip_ipi​ needs the telescope starting at time sis_isi​ and running for a length of time tit_iti​.
  • Goal: schedule as many as possible

Formally

Input:

  • Given a set PPP of projects, ∣P∣=n|P|=n∣P∣=n
  • Each request pi∈Pp_i\in Ppi​∈P occupies interval [si,fi)[s_i,f_i)[si​,fi​), where fi=si+tif_i=s_i+t_ifi​=si​+ti​

Goal: Choose a subset Π⊑P\Pi\sqsubseteq PΠ⊑P such that

  1. No two projects in Π\PiΠ have overlapping intervals.
  2. The number of selected projects ∣Π∣|\Pi|∣Π∣ is maximized.

Theorem of Greedy Strategy (Earliest Finishing Time)

Say this greedy strategy (Earliest Finishing Time) picks a set Π\PiΠ of intervals, some other strategy picks a set OOO of intervals.

Assume sorted by finishing time

  • Π={i1,i2,...,ik},∣Π∣=k\Pi=\{i_1,i_2,...,i_k\},|\Pi|=kΠ={i1​,i2​,...,ik​},∣Π∣=k
  • O={j1,j2,...,jm},∣O∣=mO=\{j_1,j_2,...,j_m\},|O|=mO={j1​,j2​,...,jm​},∣O∣=m

We want to show that ∣Π∣≥∣O∣,k>m|\Pi|\geq|O|,k>m∣Π∣≥∣O∣,k>m

Lemma: For all r<k,fir≤fjrr<k,f_{i_r}\leq f_{j_r}r<k,fir​​≤fjr​​

We proceed the proof by induction.

  • Base Case, when r=1. Π\PiΠ is the earliest finish time, and OOO cannot pick a interval with earlier finish time, so fir≤fjrf_{i_r}\leq f_{j_r}fir​​≤fjr​​

  • Inductive step, when r>1. Since Πr\Pi_rΠr​ is the earliest finish time, so for any set in OrO_rOr​, fir−1≤fjr−1f_{i_{r-1}}\leq f_{j_{r-1}}fir−1​​≤fjr−1​​, for any jrj_rjr​ inserted to OrO_rOr​, it can also be inserted to Πr\Pi_rΠr​. So OrO_rOr​ cannot pick an interval with earlier finish time than PiPiPi since it will also be picked by definition if OrO_rOr​ is the optimal solution OPTOPTOPT.

Problem of “Greedy Stays Ahead” Proof

  • Every problem has very different theorem.
  • It can be challenging to even write down the correct statement that you must prove.
  • We want a systematic approach to prove the correctness of greedy algorithms.

Road Map to Prove Greedy Algorithm

1. Make a Choice

Pick an interval based on greedy choice, say qqq

Proof: Greedy Choice Property: Show that using our first choice is not “fatal” – at least one optimal solution makes this choice.

Techniques: Exchange Argument: “If an optimal solution does not choose qqq, we can turn it into an equally good solution that does.”

Let Π∗\Pi^*Π∗ be any optimal solution for project set PPP.

  • If q∈Π∗q\in \Pi^*q∈Π∗, we are done.
  • Otherwise, let xxx be the optimal solution from Π∗\Pi^*Π∗ that does not pick qqq. We create another solution Π∗ˉ\bar{\Pi^*}Π∗ˉ that replace xxx with qqq, and prove that the Π∗ˉ\bar{\Pi^*}Π∗ˉ is as optimal as Π∗\Pi^*Π∗

2. Create a smaller instance P′P'P′ of the original problem

P′P'P′ has the same optimization criteria.

Proof: Inductive Structure: Show that after making the first choice, we’re left with a smaller version of the same problem, whose solution we can safely combine with the first choice.

Let P′P'P′ be the subproblem left after making first choice qqq in problem PPP and let Π′\Pi'Π′ be an optimal solution to P′P'P′. Then Π=Π∗∪{q}\Pi=\Pi^*\cup\{q\}Π=Π∗∪{q} is an optimal solution to PPP.

P′=P−{q}−{P'=P-\{q\}-\{P′=P−{q}−{projects conflicting with q}q\}q}

3. Solution: Union of choices that we made

Union of choices that we made.

Proof: Optimal Substructure: Show that if we solve the subproblem optimally, adding our first choice creates an optimal solution to the whole problem.

Let qqq be the first choice, P′P'P′ be the subproblem left after making qqq in problem PPP, Π′\Pi'Π′ be an optimal solution to P′P'P′. We claim that Π=Π′∪{q}\Pi=\Pi'\cup \{q\}Π=Π′∪{q} is an optimal solution to PPP.

We proceed the proof by contradiction.

Assume that Π=Π′+{q}\Pi=\Pi'+\{q\}Π=Π′+{q} is not optimal.

By Greedy choice property GCPGCPGCP. we already know that ∃\exists∃ an optimal solution Π∗\Pi^*Π∗ for problem PPP that contains qqq. If Π\PiΠ is not optimal, cost(Π∗)<cost(Π)cost(\Pi^*)<cost(\Pi)cost(Π∗)<cost(Π). Then since Π∗−q\Pi^*-qΠ∗−q is also a feasible solution to P′P'P′. cost(Π∗−q)>cost(Π−q)=Π′cost(\Pi^*-q)>cost(\Pi-q)=\Pi'cost(Π∗−q)>cost(Π−q)=Π′ which leads to contradiction that Π′\Pi'Π′ is an optimal solution to P′P'P′.

4. Put 1-3 together to write an inductive proof of the Theorem

This is independent of problem, same for every problem.

Use scheduling problem as an example:

Theorem: given a scheduling problem PPP, if we repeatedly choose the remaining feasible project with the earliest finishing time, we will construct an optimal feasible solution to PPP.

Proof: We proceed by induction on ∣P∣|P|∣P∣. (based on the size of problem PPP).

  • Base case: ∣P∣=1|P|=1∣P∣=1.
  • Inductive step.
    • Inductive hypothesis: For all problems of size <n<n<n, earliest finishing time (EFT) gives us an optimal solution.
    • EFT is optimal for problem of size nnn.
    • Proof: Once we pick q, because of greedy choice. P′=P={q}−{P'=P=\{q\} -\{P′=P={q}−{interval that conflict with q}q\}q}. ∣P′∣<n|P'|<n∣P′∣<n, By Inductive hypothesis, EFT gives us an optimal solution to P′P'P′, but by inductive substructure, and optimal substructure. Π′\Pi'Π′ (optimal solution to P′P'P′), we have optimal solution to PPP.

this step always holds as long as the previous three properties hold, and we don’t usually write the whole proof.

File compression problem

You have nnn files of different sizes fif_ifi​.

You want to merge them to create a single file. merge(fi,fj)merge(f_i,f_j)merge(fi​,fj​) takes time fi+fjf_i+f_jfi​+fj​ and creates a file of size fk=fi+fjf_k=f_i+f_jfk​=fi​+fj​.

Goal: Find the order of merges such that the total time to merge is minimized.

Thinking process: The merge process is a binary tree and each of the file is the leaf of the tree.

The total time required =∑i=1ndifi\sum^n_{i=1} d_if_i∑i=1n​di​fi​, where did_idi​ is the depth of the file in the compression tree.

So compressing the smaller file first may yield a faster run time.

Greedy Choice Property

Construct part of the solution by making a locally good decision.

Lemma: ∃\exist∃ some optimal solution that merges the two smallest file first, lets say [f1,f2][f_1,f_2][f1​,f2​]

Proof: Exchange argument

  • Case 1: Optimal choice already merges f1,f2f_1,f_2f1​,f2​, done. Time order does not matter in this problem at some point.
    • eg: [2,2,3], merge 2,3 and 2,2 first don’t change the total cost
  • Case 2: Optimal choice does not merges f1f_1f1​ and f2f_2f2​.
    • Suppose the optimal solution merges fx,fyf_x,f_yfx​,fy​ as the deepest merge.
    • Then dx≥d1,dy≥d2d_x\geq d_1,d_y\geq d_2dx​≥d1​,dy​≥d2​. Exchanging f1,f2f_1,f_2f1​,f2​ with fx,fyf_x,f_yfx​,fy​ would yield a strictly less greater solution since f1,f2f_1,f_2f1​,f2​ already smallest.

Inductive Structure

  • We can combine feasible solution to the subproblem P′P'P′ with the greedy choice to get a feasible solution to PPP
  • After making greedy choice qqq, we are left with a strictly smaller subproblem P′P'P′ with the same optimality criteria of the original problem

We proceed the proof by contradiction.

Assume that Π=Π∗+{q}\Pi=\Pi^*+\{q\}Π=Π∗+{q} is not optimal.
