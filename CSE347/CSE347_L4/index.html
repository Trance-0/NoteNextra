Maximum Flow

Example 1: Ship cement from factory to building

Input sss: source, ttt: destination

Graph with directed edges weights on each edge: capacity

Goal: Ship as much stuff as possible while obeying capacity constrains.

Graph: (V,E)(V,E)(V,E) directed and weighted

  • Unique source and sink nodes →s,t\to s, t→s,t
  • Each edge has capacity c(e)c(e)c(e) [Integer]

A valid flow assignment assigns an integer f(e)f(e)f(e) to each edge s.t.

Capacity constraint: 0≤f(e)≤c(e)0\leq f(e)\leq c(e)0≤f(e)≤c(e)

Flow conservation:

∑e∈Ein(v)f(e)=∑e∈Eout(v)f(e),∀v∈V−s,t\sum_{e\in E_{in}(v)}f(e)=\sum_{e\in E_{out}(v)}f(e),\forall v\in V-{s,t}e∈Ein​(v)∑​f(e)=e∈Eout​(v)∑​f(e),∀v∈V−s,t

Ein(v)E_{in}(v)Ein​(v): set of incoming edges to vvv Eout(v)E_{out}(v)Eout​(v): set of outgoing edges from vvv

Compute: Maximum Flow: Find a valid flow assignment to

Maximize ∣F∣=∑e∈Ein(t)f(e)=∑e∈Eout(s)f(e)|F|=\sum_{e\in E_{in}(t)}f(e)=\sum_{e\in E_{out}(s)}f(e)∣F∣=∑e∈Ein​(t)​f(e)=∑e∈Eout​(s)​f(e) (total units received by end and sent by source)

Additional assumptions

  1. sss has no incoming edges, ttt has no outgoing edges
  2. You do not have a cycle of 2 nodes

A proposed algorithm:

  1. Find a path from sss to ttt
  2. Push as much flow along the path as possible
  3. Adjust the capacities
  4. Repeat until we cannot find a path

Residual Graph: If there is an edge e=(u,v)e=(u,v)e=(u,v) in GGG, we will add a back edge eˉ=(v,u)\bar{e}=(v,u)eˉ=(v,u). Capacity of eˉ=\bar{e}=eˉ= flow on eee. Call this graph GRG_RGR​.

Algorithm:

  • Find an “augmenting path” PPP.
    • PPP can contain forward or backward edges!
  • Say the smallest residual capacity along the path is kkk.
  • Push kkk flow on the path (f(e)=f(e)+kf(e) =f(e) + kf(e)=f(e)+k for all edges on path PPP)
    • Reduce the capacity of all edges on the path PPP by kkk
    • Increase the capacity of the corresponding mirror/back edges
  • Repeat until there are no augmenting paths

Formalize: Ford-Fulkerson (FF) Algorithm

  1. Initialize the residual graph GR=GG_R=GGR​=G
  2. Find an augmenting path PPP with capacity kkk (min capacity of any edge on PPP)
  3. Fix up the residual capacities in GRG_RGR​
    • c(e)=c(e)−k,∀e∈Pc(e)=c(e)-k,\forall e\in Pc(e)=c(e)−k,∀e∈P
    • c(eˉ)=c(eˉ)+k,∀eˉ∈Pc(\bar{e})=c(\bar{e})+k,\forall \bar{e}\in Pc(eˉ)=c(eˉ)+k,∀eˉ∈P
  4. Repeat 2 and 3 until no augmenting path can be found in GRG_RGR​.
def ford_fulkerson_algo(G,n,s,t):
    """
    Args:
        G: is the graph for max_flow
        n: is the number of vertex in the graph
        s: start vertex of flow
        t: end vertex of flow
    Returns:
        the max flow in graph from s to t
    """
    # Initialize the residual graph $G_R=G$
    GR=[defaultdict(int) for i in range(n)]
    for i in range(n):
        for v,_ in enumerate(G[i]):
            # weight w is unused
            GR[v][i]=0
    path=set()
    def augP(cur):
        # Find an augumentting path $P$ with capacity $k$ (min capacity of any edge on $P$)
        if cur==t: return True
        # true for edge in residual path, false for edge in graph
        for v,w in G[cur]:
            if w==0 or (cur,v,False) in path: continue
            path.add((cur,v,False))
            if augP(v): return True
            path.remove((cur,v,False))
        for v,w in GR[cur]:
            if w==0 or (cur,v,True) in path: continue
            path.add((cur,v,True))
            if augP(v): return True
            path.remove((cur,v,True))
        return False
    while augP(s):
        k=min([GR[a][b] if isR else G[a][b] for a,b,isR in path])
        # Fix up the residual capacities in $G_R$
        # - $c(e)=c(e)-k,\forall e\in P$
        # - $c(\bar{e})=c(\bar{e})+k,\forall \bar{e}\in P$
        for a,b,isR in path:
            if isR:
                GR[a][b]+=k
            else:
                G[a][b]-=k
    return sum(GR[s].values())

Proof of Correctness: Valid Flow

Lemma 1: FF finds a valid flow

  • Capacity and conservation constrains are not violated
  • Capacity constraint: 0≤f(e)≤c(e)0\leq f(e)\leq c(e)0≤f(e)≤c(e)
  • Flow conservation: ∑e∈Ein(v)f(e)=∑e∈Eout(v)f(e),∀v∈V−{s,t}\sum_{e\in E_{in}(v)}f(e)=\sum_{e\in E_{out}(v)}f(e),\forall v\in V-\{s,t\}∑e∈Ein​(v)​f(e)=∑e∈Eout​(v)​f(e),∀v∈V−{s,t}

Proof: We proceed by induction on augmenting paths

Inductive Case

Inductive Step:

Now we find an augmented path PPP in GRGRGR, pushed kkk (which is the smallest edge capacity on PPP). Argue that the constraints are not violated.

Capacity Constrains: Consider an edge eee in PPP.

  • If eee is an forward edge (in the original graph)
    • by construction of GRG_RGR​, it had left over capacities.
  • If eee is an back edge with residual capacity ≥k\geq k≥k
    • flow on real edge reduces, but the real capacity is still ≥0\geq 0≥0, no capacity constrains violation.

Conservation Constrains: Consider a vertex vvv on path PPP

  1. Both forward edges
    • No violation, push kkk flow into vvv and out.
  2. Both back edges
    • No violation, push kkk less flow into vvv and out.
  3. Redirecting flow
    • No violation, change of 000 by k−kk-kk−k on vvv.

Bipartite graph G=(V,E)G=(V,E)G=(V,E) (unweighted and undirected)

  • Vertices are either in set LLL or RRR

Matching: A subset of edges M⊆EM\subseteq EM⊆E s.t.

Example 3: Image Segmentation: (reduction from min-cut)

Given:

  • Image consisting of an object and a background.
  • the object occupies some set of pixels AAA, while the background occupies the remaining pixels BBB.

Required:

  • Separate AAA from BBB but if doesn’t know which pixels are each.
  • For each pixel i,pii,p_ii,pi​ is the probability that i∈Ai\in Ai∈A
  • For each pair of adjacent pixels i,j,ciji,j,c_{ij}i,j,cij​ is the cost of placing the object boundary between them. i.e. putting iii in AAA and jjj in BBB.
  • A segmentation of the image is an assignment of each pixel to AAA or BBB.
  • The goal is to find a segmentation that maximizes
∑i∈Api+∑i∈B(1−pi)−∑i,j on boundarycij\sum_{i\in A}p_i+\sum_{i\in B}(1-p_i)-\sum_{i,j\ on \ boundary}c_{ij}i∈A∑​pi​+i∈B∑​(1−pi​)−i,j on boundary∑​cij​

Solution:

  • Let’s turn our maximization into a minimization
  • If the image has NNN pixels, then we can rewrite the objective as

because N=∑i∈Api+∑i∈A(1−pi)+∑i∈Bpi+∑i∈B(1−pi)N=\sum_{i\in A}p_i+\sum_{i\in A}(1-p_i)+\sum_{i\in B}p_i+\sum_{i\in B}(1-p_i)N=∑i∈A​pi​+∑i∈A​(1−pi​)+∑i∈B​pi​+∑i∈B​(1−pi​) boundary

New maximization problem:

Max(N−∑i∈A(1−pi)−∑i∈Bpi−∑i,j on boundarycij)Max\left( N-\sum_{i\in A}(1-p_i)-\sum_{i\in B}p_i-\sum_{i,j\ on \ boundary}c_{ij}\right)Max​N−i∈A∑​(1−pi​)−i∈B∑​pi​−i,j on boundary∑​cij​​

Now, this is equivalent ot minimizing

∑i∈A(1−pi)+∑i∈Bpi+∑i,j on boundarycij\sum_{i\in A}(1-p_i)+\sum_{i\in B}p_i+\sum_{i,j\ on \ boundary}c_{ij}i∈A∑​(1−pi​)+i∈B∑​pi​+i,j on boundary∑​cij​

Second steps

  • Form a graph with nnn vertices, viv_ivi​ on for each pixel
  • Add vertices sss and ttt
  • For each viv_ivi​, add edges S−TS-TS−T cut of GGG assigned each viv_ivi​ to either SSS side or TTT side.
  • The SSS side of an S−TS-TS−T is the AAA side, while the TTT side of the cur is the BBB side.
  • Observer that if viv_ivi​ goes on the SSS side, it becomes part of AAA, so the cut increases by 1−p1-p1−p. Otherwise, it become part of BBB, so the cut increases by pip_ipi​ instead.
  • Now add edges vi→vjv_i\to v_jvi​→vj​ with capacity cijc_{ij}cij​ for all adjacent pixels pairs i,ji,ji,j
  • If viv_ivi​ and vjv_jvj​ end up on opposite sides of the cut (boundary), then the cut increases by cijc_{ij}cij​.
  • Conclude that any S−TS-TS−T cut that assigns S⊆VS\subseteq VS⊆V to the AAA side and V\SV\backslash SV\S to the BBB side pays a total of
    1. 1−pi1-p_i1−pi​ for each viv_ivi​ on the AAA side
    2. pip_ipi​ for each viv_ivi​ on the BBB side
    3. cijc_{ij}cij​ for each adjacent pair i,ji,ji,j that is at the boundary. i.e. i∈S and j∈V\Si\in S\ and\ j\in V\backslash Si∈S and j∈V\S
  • Conclude that a cut with a capacity ccc implies a segmentation with objective value cscscs.
  • The converse can (and should) be also checked: a segmentation with subjective value ccc implies a S−TS-TS−T cut with capacity ccc.

Algorithm

  • Given an image with NNN pixels, build the graph GGG as desired.
  • Use the FF algorithm to find a minimum S−TS-TS−T cut of GGG
  • Use this cut to assign each pixel to AAA or BBB as described, i.e pixels that correspond to vertices on the SSS side are assigned to AAA and those corresponding to vertices on the TTT side to BBB.
  • Minimizing the cut capacity minimizes our transformed minimization objective function.

Running time

The graph GGG contains Θ(N)\Theta(N)Θ(N) edges, because each pixel is adjacent to a maximum of of 4 neighbors and SSS and TTT.

FF algorithm has running time O((m+n)∣F∣)O((m+n)|F|)O((m+n)∣F∣), where ∣F∣≤∣n∣|F|\leq |n|∣F∣≤∣n∣ is the size of set of min-cut. The edge count is m=6nm=6nm=6n.

So the total running time is O(n2)O(n^2)O(n2)
