Reductions

We say that a problem AAA reduces to a problem BBB if there is a polynomial time reduction function fff such that for all xxx, x∈A  ⟺  f(x)∈Bx \in A \iff f(x) \in Bx∈A⟺f(x)∈B.

To prove a reduction, we need to show that the reduction function fff:

  1. runs in polynomial time
  2. x∈A  ⟺  f(x)∈Bx \in A \iff f(x) \in Bx∈A⟺f(x)∈B.

Useful results from reductions

  1. BBB is at least as hard as AAA if A≤BA \leq BA≤B.
  2. If we can solve BBB in polynomial time, then we can solve AAA in polynomial time.
  3. If we want to solve problem AAA, and we already know an efficient algorithm for BBB, then we can use the reduction A≤BA \leq BA≤B to solve AAA efficiently.
  4. If we want to show that BBB is NP-hard, we can do this by showing that A≤BA \leq BA≤B for some known NP-hard problem AAA.

PPP is the class of problems that can be solved in polynomial time. NPNPNP is the class of problems that can be verified in polynomial time.

We know that P⊆NPP \subseteq NPP⊆NP.

NP-complete problems

A problem is NP-complete if it is in NPNPNP and it is also NP-hard.

NP

A problem is in NPNPNP if

  1. there is a polynomial size certificate for the problem, and
  2. there is a polynomial time verifier for the problem that takes the certificate and checks whether it is a valid solution.

NP-hard

A problem is NP-hard if every instance of NPNPNP hard problem can be reduced to it in polynomial time.

List of known NP-hard problems:

  1. 3-SAT (or SAT):
    • Statement: Given a boolean formula in CNF with at most 3 literals per clause, is there an assignment of truth values to the variables that makes the formula true?
  2. Independent Set:
    • Statement: Given a graph GGG and an integer kkk, does GGG contain a set of kkk vertices such that no two vertices in the set are adjacent?
  3. Vertex Cover:
    • Statement: Given a graph GGG and an integer kkk, does GGG contain a set of kkk vertices such that every edge in GGG is incident to at least one vertex in the set?
  4. 3-coloring:
    • Statement: Given a graph GGG, can each vertex be assigned one of 3 colors such that no two adjacent vertices have the same color?
  5. Hamiltonian Cycle:
    • Statement: Given a graph GGG, does GGG contain a cycle that visits every vertex exactly once?
  6. Hamiltonian Path:
    • Statement: Given a graph GGG, does GGG contain a path that visits every vertex exactly once?

Approximation Algorithms

  • Consider optimization problems whose decision problem variant is NP-hard. Unless P=NP, finding an optimal solution to these problems cannot be done in polynomial time.
  • In approximation algorithms, we make a trade-o↵: we’re willing to accept sub-optimal solutions in exchange for polynomial runtime.
  • The Approximation Ratio of our algorithm is the worst-case ratio of our solution to the optimal solution.
  • For minimization problems, this ratio is max⁡l∈L(cA(l)cOPT(l))\max_{l\in L}\left(\frac{c_A(l)}{c_{OPT}(l)}\right)maxl∈L​(cOPT​(l)cA​(l)​), since our solution will be larger than OPT.
  • For maximization problems, this ratio is min⁡l∈L(cOPT(l)cA(l))\min_{l\in L}\left(\frac{c_{OPT}(l)}{c_A(l)}\right)minl∈L​(cA​(l)cOPT​(l)​), since our solution will be smaller than OPT.
  • If given an algorithm, and you need to show it has some desired approximation ratio, there are a few approaches.
  • In recitation, we saw Max-Subset Sum. We found upper bounds on the optimal solution and showed that the given algorithm would always give a solution with value at least half of the upper bound, giving our approximation ratio of 2.
  • In lecture, you saw the Vertex Cover 2-approximation. Here, you would select any uncovered edge (u,v)(u, v)(u,v) and add both u and v to the cover. We argued that at least one of u or v must be in the optimal cover, as the edge must be covered, so at every step we added at least one vertex from an optimal solution, and potentially one extra. So, the size of our cover could not be any larger than twice the optimal.

Randomized Algorithms

Sometimes, we can get better expected performance from an algorithm by introducing randomness.

We make the tradeoff guarantee runtime and solution quality from a deterministic algorithm, to expected runtime and quality from randomized algorithms.

We can make various bounds and tricks to calculate and amplify the probability of succeeding.

Online Algorithms

  • We make decisions on the fly, without knowing the future.
  • The offline optimum is the optimal solution that knows the future.
  • The competitive ratio of an online algorithm is the worst-case ratio of the cost of the online algorithm to the cost of the offline optimum.
