Offline Reinforcement Learning: Introduction and Challenges

Offline reinforcement learning (offline RL), also called batch RL, aims to learn an optimal policy -without- interacting with the environment. Instead, the agent is given a fixed dataset of transitions collected by an unknown behavior policy.

The Offline RL Dataset

We are given a static dataset:

D=(si,ai,s′−i,ri)−i=1ND = { (s_i, a_i, s'-i, r_i ) }-{i=1}^ND=(si​,ai​,s′−i,ri​)−i=1N

Parameter explanations:

  • sis_isi​: state sampled from behavior policy state distribution.
  • aia_iai​: action selected by the behavior policy πbeta\pi_betaπb​eta.
  • si′s'_isi′​: next state sampled from environment dynamics p(s′∣s,a)p(s'|s,a)p(s′∣s,a).
  • rir_iri​: reward observed for transition (si,ai)(s_i,a_i)(si​,ai​).
  • NNN: total number of transitions in the dataset.
  • DDD: full offline dataset used for training.

The goal is to learn a new policy π\piπ maximizing expected discounted return using only DDD:

max⁡π;E[∑t=0Tγtr(st,at)]\max_{\pi} ; \mathbb{E}\Big[\sum_{t=0}^T \gamma^t r(s_t, a_t)\Big]πmax​;E[t=0∑T​γtr(st​,at​)]

Parameter explanations:

  • π\piπ: policy we want to learn.
  • r(s,a)r(s,a)r(s,a): reward received for state-action pair.
  • γ\gammaγ: discount factor controlling weight of future rewards.
  • TTT: horizon or trajectory length.

Why Offline RL Is Difficult

Offline RL is fundamentally harder than online RL because:

  • The agent cannot try new actions to fix wrong value estimates.
  • The policy may choose out-of-distribution actions not present in DDD.
  • Q-value estimates for unseen actions can be arbitrarily incorrect.
  • Bootstrapping on wrong Q-values can cause divergence.

This leads to two major failure modes:

  1. —Distribution shift—: new policy actions differ from dataset actions.
  2. —Extrapolation error—: the Q-function guesses values for unseen actions.

Extrapolation Error Problem

In standard Q-learning, the Bellman backup is:

Q(s,a)←r+γmax⁡a′Q(s′,a′)Q(s,a) \leftarrow r + \gamma \max_{a'} Q(s', a')Q(s,a)←r+γa′max​Q(s′,a′)

Parameter explanations:

  • Q(s,a)Q(s,a)Q(s,a): estimated value of taking action aaa in state sss.
  • max⁡a′\max_{a'}maxa′​: maximum over possible next actions.
  • a′a'a′: candidate next action for evaluation in backup step.

If a′a'a′ was rarely or never taken in the dataset, Q(s′,a′)Q(s',a')Q(s′,a′) is poorly estimated, so Q-learning boots off invalid values, causing instability.

Behavior Cloning (BC): The Safest Baseline

The simplest offline method is to imitate the behavior policy:

max⁡ϕ;E(s,a)∼D[log⁡πϕ(a∣s)]\max_{\phi} ; \mathbb{E}_{(s,a) \sim D}[\log \pi_{\phi}(a|s)]ϕmax​;E(s,a)∼D​[logπϕ​(a∣s)]

Parameter explanations:

  • ϕ\phiϕ: neural network parameters of the cloned policy.
  • πϕ\pi_{\phi}πϕ​: learned policy approximating behavior policy.
  • log⁡πϕ(a∣s)\log \pi_{\phi}(a|s)logπϕ​(a∣s): negative log-likelihood loss.

Pros:

  • Does not suffer from extrapolation error.
  • Extremely stable.

Cons:

  • Cannot outperform the behavior policy.
  • Ignores reward information entirely.

Naive Offline Q-Learning Fails

Directly applying off-policy Q-learning on DDD generally leads to:

  • Overestimation of unseen actions.
  • Divergence due to extrapolation error.
  • Policies worse than behavior cloning.

Strategies for Safe Offline RL

There are two primary families of solutions:

  1. —Policy constraint methods—
  2. —Conservative value estimation methods—

1. Policy Constraint Methods

These methods restrict the learned policy to stay close to the behavior policy so it does not take unsupported actions.

Advantage Weighted Regression (AWR / AWAC)

Policy update:

π(a∣s)∝πbeta(a∣s)exp⁡(1λA(s,a))\pi(a|s) \propto \pi_{beta}(a|s)\exp\left(\frac{1}{\lambda}A(s,a)\right)π(a∣s)∝πbeta​(a∣s)exp(λ1​A(s,a))

Parameter explanations:

  • πbeta\pi_{beta}πbeta​: behavior policy used to collect dataset.
  • A(s,a)A(s,a)A(s,a): advantage function derived from Q or V estimates.
  • λ\lambdaλ: temperature controlling strength of advantage weighting.
  • exp⁡(⋅)\exp(\cdot)exp(⋅): positive weighting on high-advantage actions.

Properties:

  • Uses advantages to filter good and bad actions.
  • Improves beyond behavior policy while staying safe.

Batch-Constrained Q-learning (BCQ)

BCQ constrains the policy using a generative model:

  1. Train a VAE GωG_{\omega}Gω​ to model aaa given sss.
  2. Train a small perturbation model ξ\xiξ.
  3. Limit the policy to a=Gω(s)+ξ(s)a = G_{\omega}(s) + \xi(s)a=Gω​(s)+ξ(s).

Parameter explanations:

  • Gω(s)G_{\omega}(s)Gω​(s): VAE-generated action similar to data actions.
  • ω\omegaω: VAE parameters.
  • ξ(s)\xi(s)ξ(s): small correction to generated actions.
  • aaa: final policy action constrained near dataset distribution.

BCQ avoids selecting unseen actions and strongly reduces extrapolation.

BEAR (Bootstrapping Error Accumulation Reduction)

BEAR adds explicit constraints:

DMMD(π(a∣s),πbeta(a∣s))<ϵD_{MMD}\left(\pi(a|s), \pi_{beta}(a|s)\right) < \epsilonDMMD​(π(a∣s),πbeta​(a∣s))<ϵ

Parameter explanations:

  • DMMDD_{MMD}DMMD​: Maximum Mean Discrepancy distance between action distributions.
  • ϵ\epsilonϵ: threshold restricting policy deviation from behavior policy.

BEAR controls distribution shift more tightly than BCQ.

2. Conservative Value Function Methods

These methods modify Q-learning so Q-values of unseen actions are -underestimated-, preventing the policy from exploiting overestimated values.

Conservative Q-Learning (CQL)

One formulation is:

J(Q)=JTD(Q)+α(Ea∼π(⋅∣s)Q(s,a)−Ea∼DQ(s,a))J(Q) = J_{TD}(Q) + \alpha\big(\mathbb{E}_{a\sim\pi(\cdot|s)}Q(s,a) - \mathbb{E}_{a\sim D}Q(s,a)\big)J(Q)=JTD​(Q)+α(Ea∼π(⋅∣s)​Q(s,a)−Ea∼D​Q(s,a))

Parameter explanations:

  • JTDJ_{TD}JTD​: standard Bellman TD loss.
  • α\alphaα: weight of conservatism penalty.
  • Ea∼π(⋅∣s)\mathbb{E}_{a\sim\pi(\cdot|s)}Ea∼π(⋅∣s)​: expectation over policy-chosen actions.
  • Ea∼D\mathbb{E}_{a\sim D}Ea∼D​: expectation over dataset actions.

Effect:

  • Increases Q-values of dataset actions.
  • Decreases Q-values of out-of-distribution actions.

IQL avoids constraints entirely by using expectile regression:

Value regression:

V(s)=arg⁡min⁡v;E[ρτ(Q(s,a)−v)]V(s) = \arg\min_{v} ; \mathbb{E}\big[\rho_{\tau}(Q(s,a) - v)\big]V(s)=argvmin​;E[ρτ​(Q(s,a)−v)]

  • vvv: scalar value estimate for state sss.
  • ρτ(x)\rho_{\tau}(x)ρτ​(x): expectile regression loss.

  • For τ<1\tau < 1τ<1, IQL reduces sensitivity to large (possibly incorrect) Q-values.
  • Implicitly conservative without special constraints.

IQL often achieves state-of-the-art performance due to simplicity and stability.

Model-Based Offline RL

Forward Model-Based RL

Train a dynamics model:

pθ(s′∣s,a)p_{\theta}(s'|s,a)pθ​(s′∣s,a)

Parameter explanations:

  • pθp_{\theta}pθ​: learned transition model.
  • θ\thetaθ: parameters of transition model.

We can generate synthetic transitions using pθp_{\theta}pθ​, but model error accumulates.

rmodel(s,a)=r(s,a)−β,u(s,a)r_{model}(s,a) = r(s,a) - \beta , u(s,a)rmodel​(s,a)=r(s,a)−β,u(s,a)

Reverse Model-Based Imagination (ROMI)

ROMI generates new training data by -backward- imagination.

Reverse Dynamics Model

ROMI learns:

pψ(st∣st+1,at)p_{\psi}(s_{t} \mid s_{t+1}, a_{t})pψ​(st​∣st+1​,at​)

Parameter explanations:

  • ψ\psiψ: parameters of reverse dynamics model.
  • st+1s_{t+1}st+1​: later state.
  • ata_{t}at​: action taken leading to st+1s_{t+1}st+1​.
  • sts_{t}st​: predicted predecessor state.

ROMI also learns a reverse policy for sampling likely predecessor actions.

Reverse Imagination Process

Given a goal state sgs_{g}sg​:

  1. Sample ata_{t}at​ from reverse policy.
  2. Predict sts_{t}st​ from reverse dynamics.
  3. Form imagined transition (st,at,st+1)(s_{t}, a_{t}, s_{t+1})(st​,at​,st+1​).
  4. Repeat to build longer imagined trajectories.

Benefits:

  • Imagined transitions end in real states, ensuring grounding.
  • Completes missing parts of dataset.
  • Helps propagate reward backward reliably.

ROMI combined with conservative RL often outperforms standard offline methods.

Summary of Lecture 22

Offline RL requires balancing:

  • Improvement beyond dataset behavior.
  • Avoiding unsafe extrapolation to unseen actions.

Three major families of solutions:

  1. Policy constraints (BCQ, BEAR, AWR)
  2. Conservative Q-learning (CQL, IQL)
  3. Model-based conservatism and imagination (MOPO, MOReL, ROMI)

Offline RL is becoming practical for real-world domains such as healthcare, robotics, autonomous driving, and recommender systems.
