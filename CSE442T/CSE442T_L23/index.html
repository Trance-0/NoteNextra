Chapter 7: Composability

Zero-knowledge proofs

Let the Prover Peggy and the Verifier Victor.

Peggy wants to prove to Victor that she knows a secret xxx without revealing anything about xxx. (e.g. xxx such that gx=ymod  pg^x=y\mod pgx=ymodp)

Zero-knowledge proofs protocol

The protocol should satisfy the following properties:

  • Completeness: If Peggy knows xxx, she can always make Victor accept.
  • Soundness: If a malicious Prover P∗P^*P∗ does not know xxx, then VVV accepts with probability at most ϵ(n)\epsilon(n)ϵ(n).
  • Zero-knowledge: After the process, V∗V^*V∗ (possibly dishonest Verifier) knows no more about xxx than he did before.

[The interaction could have been faked without PPP]

Example: Hair counting magician

“Magician” who claims they can count the number of hairs on your head.

secret info: the method of counting.

Repeat the following process for kkk times:

  1. “Magician” tells the number of hairs
  2. You remove some hair b∈{0,1}b\in \{0,1\}b∈{0,1} from your head.
  3. “Magician” tells the number of hairs left.
  4. Reject if the number of hairs is incorrect. Accept after kkk times. (to our desired certainty)

Definition

Let PPP and VVV be two interactive Turing machines.

Let xxx be the shared input, yyy be the secret knowledge, zzz be the existing knowledge about yyy, with r1,r2,⋯ ,rkr_1,r_2,\cdots,r_kr1​,r2​,⋯,rk​ being the random tapes.

VVV should output accept or reject after the interaction for qqq times.

class P(TuringMachine):
    """
    :param x: the shared input with V
    :param y: auxiliary input (the secret knowledge)
    :param z: auxiliary input (could be existing knowledge about y)
    :param r_i: random message
    """
    def run(self, x)->str:
        """
        :return: the message to be sent to V $m_p$
        """
 
class V(TuringMachine):
    """
    The verifier will output accept or reject after the interaction for $q$ times.
 
    :param x: the shared input with P
    :param y: auxiliary input (the secret knowledge)
    :param z: auxiliary input (could be existing knowledge about y)
    :param r_i: random message
    """
    def run(self, q: int)->bool:
        """
        :param q: the number of rounds
        :return: accept or reject
        """
        for i in range(q):
            m_v = V.run(i)
            m_p = P.run(m_v)
            if m_p!=m_v:
                return False
        return True

Let the transcript be the sequence of messages exchanged between PPP and VVV. Transcript=(m1p,m1v,m2p,m2v,⋯ ,mqp,mqv)\text{Transcript} = (m_1^p,m_1^v,m_2^p,m_2^v,\cdots,m_q^p,m_q^v)Transcript=(m1p​,m1v​,m2p​,m2v​,⋯,mqp​,mqv​).

Define (P,V)(P,V)(P,V) be the zero-knowledge proof protocol. For a language LLL, (P,V)(P,V)(P,V) is a zero-knowledge proof for LLL if:

{Transcript[P(x,y)↔V∗(x,z)∣x∈L,y←{0,1}n]}and{S(x,z)∣x∉L}.\{\text{Transcript}[P(x,y)\leftrightarrow V^*(x,z)\mid x\in L,y\leftarrow \{0,1\}^n]\}\quad\text{and}\quad\{S(x,z)\mid x\notin L\}.{Transcript[P(x,y)↔V∗(x,z)∣x∈L,y←{0,1}n]}and{S(x,z)∣x∈/L}.

Example: Graph isomorphism

Let G0G_0G0​ and G1G_1G1​ be two graphs.

VVV picks a random permutation π∈Sn\pi\in S_nπ∈Sn​ and sends GπG_\piGπ​ to PPP.

PPP needs to determine if Gπ=G0G_\pi=G_0Gπ​=G0​ or Gπ=G1G_\pi=G_1Gπ​=G1​.

If they are isomorphic, then ∃\exists∃ permutation σ:{1,⋯ ,n}→{1,⋯ ,n}\sigma:\{1,\cdots,n\}\rightarrow \{1,\cdots,n\}σ:{1,⋯,n}→{1,⋯,n} such that G0={(i,j)∣(i,j)∈G1}G_0=\{(i,j)\mid (i,j)\in G_1\}G0​={(i,j)∣(i,j)∈G1​}.

Protocol:

Shared input x‾=(G0,G1)\overline{x}=(G_0,G_1)x=(G0​,G1​) witness y‾=σ\overline{y}=\sigmay​=σ. Repeat the following process for nnn times, where nnn is the number of vertices.

  1. PPP picks a random permutation π∈Pn\pi\in \mathbb{P}_nπ∈Pn​ and sends Gπ=π(G0)G_\pi=\pi(G_0)Gπ​=π(G0​) to VVV.
  2. VVV picks a random b∈{0,1}b\in \{0,1\}b∈{0,1} and sends bbb to PPP.
  3. If b=1b=1b=1, PPP sends σ=π−1\sigma=\pi^{-1}σ=π−1 to VVV.
  4. If b=0b=0b=0, PPP sends σ=π\sigma=\piσ=π to VVV.
  5. VVV receives ϕ\phiϕ and checks if b=0b=0b=0 and Gσ=ϕ(G0)G_\sigma=\phi(G_0)Gσ​=ϕ(G0​) or b=1b=1b=1 and Gσ=ϕ(G1)G_\sigma =\phi(G_1)Gσ​=ϕ(G1​). Return accept if true.

If they are not isomorphic, PPP rejects with probability 1.

If they are isomorphic, PPP accepts with probability 1n!\frac{1}{n!}n!1​.

Proof:

  • Completeness: If G0G_0G0​ and G1G_1G1​ are isomorphic, then PPP can always find a permutation σ\sigmaσ such that Gσ=G0G_\sigma=G_0Gσ​=G0​ or Gσ=G1G_\sigma=G_1Gσ​=G1​.
  • Soundness:
    • If P∗P^*P∗ knows that VVV was going to send b=0b=0b=0, then they will pick Π\PiΠ and send G=Π(G0)G=\Pi(G_0)G=Π(G0​) to VVV. However, if we thought they would send 000 but they sent 111, then G=Π(G1)G=\Pi(G_1)G=Π(G1​) and they would reject.
    • If P∗P^*P∗ knows that VVV was going to send b=1b=1b=1, then they will pick Π\PiΠ and send G=Π(G1)G=\Pi(G_1)G=Π(G1​) to VVV. However, if we thought they would send 111 but they sent 000, then G=Π(G0)G=\Pi(G_0)G=Π(G0​) and they would reject.
    • The key is that P∗P^*P∗ can only response correctly with probability at most 12\frac{1}{2}21​ each time.
