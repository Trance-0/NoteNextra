Block ciphers

  1. Operate on PT one block at a time
  2. Use same key for multiple blocks (with caveats)
  3. Chaining modes intertwine successive blocks of CT (or not)

Security abstraction

View cipher as a Pseudo-Random Permutation (PRP)

PRP Definition

Defined over (K,X)(K,X)(K,X):

E:K×X→XE : K \times X \to XE:K×X→X

Such that:

  1. Efficient deterministic algorithm to evaluate E(k,x)E(k,x)E(k,x)
  2. E(k,⋅)E(k,\cdot)E(k,⋅) is one-to-one
  3. Efficient inversion algorithm D(k,y)D(k,y)D(k,y) exists

i.e., a PRF that is an invertible one-to-one mapping from message space to message space

Block cipher constructions

Feistel network

Given:

f1,…,fd:{0,1}n→{0,1}nf_1, \dots, f_d : \{0,1\}^n \to \{0,1\}^nf1​,…,fd​:{0,1}n→{0,1}n

Build invertible function:

F:{0,1}2n→{0,1}2nF : \{0,1\}^{2n} \to \{0,1\}^{2n}F:{0,1}2n→{0,1}2n

Let input be split into (L0,R0)(L_0, R_0)(L0​,R0​).

Thus Feistel is invertible regardless of whether fif_ifi​ is invertible.

Luby–Rackoff Theorem (1985)

If fff is a secure PRF, then 3-round Feistel is a secure PRP.

DES (Data Encryption Standard) — 1976

  • 16-round Feistel network
  • 64-bit block size
  • 56-bit key
  • Round functions:
fi(x)=F(ki,x)f_i(x) = F(k_i, x)fi​(x)=F(ki​,x)

Round function uses:

  • S-box (substitution box) — non-linear
  • P-box (permutation box)

To invert: use keys in reverse order.

Problem: 56-bit keyspace too small today (brute-force feasible).

Substitution–Permutation Network (SPN)

Rounds of:

  • Substitution (S-box layer)
  • Permutation (P-layer)
  • XOR with round key

All layers invertible.

AES (Advanced Encryption Standard) — 2000

  • 10 substitution-permutation rounds (128-bit key version)
  • 128-bit block size

Each round includes:

  • ByteSub (1-byte S-box)
  • ShiftRows
  • MixColumns
  • AddRoundKey

Key sizes:

  • 128-bit
  • 192-bit
  • 256-bit

Currently de facto standard symmetric-key cipher (e.g. TLS/SSL).

Block cipher modes

Challenge

Encrypt PTs longer than one block using same key while maintaining security.

ECB (Electronic Codebook)

Encrypt blocks independently:

ci=E(k,mi)c_i = E(k, m_i)ci​=E(k,mi​)

Problem:

If m1=m2m_1 = m_2m1​=m2​, then:

c1=c2c_1 = c_2c1​=c2​

Not semantically secure.

Formal non-security argument

Two-block challenge:

  • Adversary submits:
    • m0="Hello World"m_0 = \text{"Hello World"}m0​="Hello World"
    • m1="Hello Hello"m_1 = \text{"Hello Hello"}m1​="Hello Hello"
  • If c1=c2c_1 = c_2c1​=c2​, output 0; else 1

Advantage = 1

CPA model (Chosen Plaintext Attack)

Attacker:

  • Sees many PT/CT pairs under same key
  • Can submit arbitrary PTs

Definition:

AdvCPA[A,E]=∣Pr⁡[EXP(0)=1]−Pr⁡[EXP(1)=1]∣\text{Adv}_{CPA}[A,E] = \left| \Pr[\text{EXP}(0)=1] - \Pr[\text{EXP}(1)=1] \right|AdvCPA​[A,E]=∣Pr[EXP(0)=1]−Pr[EXP(1)=1]∣

Must be negligible.

ECB fails CPA security.

Moral

If same secret key is used multiple times, given same PT twice, encryption must produce different CT outputs.

Secure block modes

Idea

Augment key with:

  • Per-block nonce
  • Or chaining data from prior blocks

CBC (Cipher Block Chaining)

c1=E(k,m1⊕IV)c_1 = E(k, m_1 \oplus IV)c1​=E(k,m1​⊕IV) ci=E(k,mi⊕ci−1)c_i = E(k, m_i \oplus c_{i-1})ci​=E(k,mi​⊕ci−1​)

IV must be random/unpredictable.

CFB (Cipher Feedback)

Uses previous ciphertext as input feedback into block cipher.

OFB (Output Feedback)

si=E(k,si−1)s_i = E(k, s_{i-1})si​=E(k,si−1​) ci=mi⊕sic_i = m_i \oplus s_ici​=mi​⊕si​

Can pre-compute keystream.

Acts like stream cipher.

CTR (Counter Mode)

ci=mi⊕E(k,nonce∥counteri)c_i = m_i \oplus E(k, \text{nonce} \| \text{counter}_i)ci​=mi​⊕E(k,nonce∥counteri​)

Encryption and decryption parallelizable.

Nonce must be unique.

GCM (Galois Counter Mode)

  • Most popular (“AES-GCM”)
  • Provides authenticated encryption
  • Confidentiality + integrity

  • Nonces must be distinct

AdvnCPA[A,E]=∣Pr⁡[EXP(0)=1]−Pr⁡[EXP(1)=1]∣\text{Adv}_{nCPA}[A,E] = \left| \Pr[\text{EXP}(0)=1] - \Pr[\text{EXP}(1)=1] \right|AdvnCPA​[A,E]=∣Pr[EXP(0)=1]−Pr[EXP(1)=1]∣

In practice:

  • CBC: IV must be random
  • CTR/GCM: nonce must be unique but not necessarily random

Symmetric Encryption Summary

Stream Ciphers

  • Rely on secure PRG
  • No key re-use
  • Fast
  • Low memory
  • Less robust
  • No built-in integrity

Block Ciphers

  • Rely on secure PRP
  • Allow key re-use across blocks (secure mode required)
  • Provide authenticated encryption in some modes (e.g. GCM)
  • Slower
  • Higher memory
  • More robust
  • Used in most practical secure systems (e.g. TLS)
