Public-key building block: Trapdoor function (TDF)

Definition of trapdoor function

A trapdoor function X→YX\to YX→Y is a triple of efficient algorithms (G,F,F−1)(G,F,F^{-1})(G,F,F−1) such that:

  • G(∘)G(\circ)G(∘) is randomized algorithm outputs a key pair (pk,sk)(pk,sk)(pk,sk).
  • F(pk,∘)F(pk,\circ)F(pk,∘) is a deterministic algorithm that takes as input a public key pkpkpk and a message mmm and outputs a ciphertext ccc.
  • F−1(sk,∘)F^{-1}(sk,\circ)F−1(sk,∘) is a deterministic algorithm that takes as input a secret key sksksk and a ciphertext ccc and outputs a message mmm.

more precisely: ∀(pk,sk)\forall(pk,sk)∀(pk,sk) outputs by GGG, ∀x∈X:F−1(sk,F(pk,x))=x\forall x\in X: F^{-1}(sk,F(pk,x))=x∀x∈X:F−1(sk,F(pk,x))=x.

RSA cryptosystem: challenge

  • The implementation of the RSA cryptosystem requires various algorithms.

  • Overall

    • Representation of integers of arbitrarily large size and arithmetic operations on them

  • Encryption

    • Modular power

  • Decryption

    • Modular power

  • Setup

    • Generation of random numbers with a given number of bits (to generate candidates ppp and qqq)
    • Primality testing (to check that candidates ppp and qqq are prime)
    • Computation of the GCD (to verify that eee and φ(n)\varphi(n)φ(n) are relatively prime)
    • Computation of the multiplicative inverse (to compute ddd from eee)

Diffie-Hellman key exchange

Based on hardness of “discrete log problem”:

Given ppp, ggg, y=gx(modp)y=g^x \pmod py=gx(modp), what is xxx?

  • Eavesdropper sees: ppp, ggg, A=ga(modp)A=g^a \pmod pA=ga(modp), and B=gb(modp)B=g^b \pmod pB=gb(modp).
  • How hard is it to compute gab(modp)g^{ab} \pmod pgab(modp)?
  • More generally: define DHg(ga,gb)=gab(modp)DH_g(g^a, g^b) = g^{ab} \pmod pDHg​(ga,gb)=gab(modp).

Elliptic Curve Cryptography (ECC)

  • Parameters: curve, modulus, initial point
    • Curve: y2=x3+ax2+bx+cy^2 = x^3 + ax^2 + bx + cy2=x3+ax2+bx+c
    • Modulus: large prime number
    • Initial point: large (x,y)(x, y)(x,y)
  • Operations: addition, point doubling, dot (see tutorial)
    • Repeated addition ∼\sim∼ multiplication
    • Point doubling ∼\sim∼ multiplying by 222
    • Repeated point doubling ∼\sim∼ multiplying by powers of 222

Hard problem: analogue of discrete-log problem using elliptic curves in particular geometric space

  • See ArsTechnica tutorial, or many videos online
  • Reversing the dot and point-doubling operators in the finite field defined by the curve and modulus
  • Example: Let the finite field be defined by y2=x3+7(mod31)y^2 = x^3 + 7 \pmod{31}y2=x3+7(mod31) with initial point (x,y)(x, y)(x,y).
    • Question: Suppose we see a new point (x2,y2)(x_2, y_2)(x2​,y2​) and we know (x2,y2)=n⋅(x,y)(x_2, y_2) = n \cdot (x, y)(x2​,y2​)=n⋅(x,y). What is nnn?
    • I.e., how many times must we add (x,y)(x, y)(x,y) to itself to get (x2,y2)(x_2, y_2)(x2​,y2​)?
    • Public key: (x2,y2)(x_2, y_2)(x2​,y2​) and parameters of the ECC system
    • Private key: nnn
    • Encryption: embed message as points on the EC, run EC ops on them

Public-key encryption from TDFs

Security Theorem:

  • If (G,F,F−1)(G, F, F^{-1})(G,F,F−1) is a secure trapdoor function (TDF),
  • (Es,Ds)(E_s, D_s)(Es​,Ds​) provides authenticated encryption,
  • and H:X→KH : X \to KH:X→K is modeled as a random oracle (RO),

then (G,E,D)(G, E, D)(G,E,D) is CCARO_{\text{RO}}RO​ secure.

  • That is, it is CCA-secure in the random oracle model.
  • An additional extension is required to obtain full CCA security in the standard model, and such constructions are known.

Summary

Wrapup: symmetric vs. asymmetric systems

  1. Symmetric: faster, but key distribution hard
  2. Asymmetric: slower, but key distribution/management easier
  3. Application: secure web sessions (e.g. online shopping visit)
    1. Use symmetric-key-encrypted sessions
    2. Exchange symmetric keys with asymmetric scheme
    3. Authenticate public keys (using PKI or web of trust)
